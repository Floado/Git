Ініціалізація та клонування репозиторію . Робота з файлами

git init - Створює новий локальний репозиторій в поточному каталозі. Використовується, коли ви хочете почати відстежувати зміни в новому проекті.
git clone <адреса_репозиторію> - Створює локальну копію віддаленого репозиторію. Використовується, коли ви хочете почати працювати над існуючим проектом.
git add <файл або папка> - Додає зміни у файлі до індексу, готуючи їх до коміту. Використовується, коли ви хочете зберегти зміни перед створенням нового коміту.
git commit -m "Коментар до коміту" - Створює новий коміт з усіма змінами, доданими до індексу. Коментар описує зміни, внесені в цьому коміті.
git status - Показує стан робочого каталогу та індексу, включаючи змінені, нові та видалені файли. Використовується для перевірки, які зміни ще не були закомічені.












Робота з гілками. Синхронізація з віддаленим репозиторієм

git branch - Переглядає список всіх гілок у репозиторії.
git branch <ім'я_гілки> - Створює нову гілку.
git checkout <ім'я_гілки> - Перемикається на іншу гілку.
git push <віддалений_репозиторій> <гілка> - Відправляє локальні коміти на віддалений репозиторій.
git pull <віддалений_репозиторій> <гілка> - Завантажує нові коміти з віддаленого репозиторію та об'єднує їх з вашою локальною гілкою.
